# Xray-script-portable 项目工作原理

## 项目概述

Xray-script-portable 是一个基于 Bash 的自动化脚本项目，用于在 Linux 服务器（Ubuntu 16+、Debian 9+、CentOS 7+）上**一键部署和管理 Xray-core 代理服务**。项目支持多种 XTLS 协议配置（Vision、XHTTP、mKCP、Trojan、Fallback、SNI），集成 Nginx 反向代理、SSL 证书管理、Cloudflare WARP 代理以及路由规则管理等功能，并提供完整的多语言（中文/英文）交互式菜单界面。

## 整体架构

```
┌──────────────────────────────────────────────────────┐
│                    install.sh (入口)                  │
│  系统检测 → 依赖安装 → 项目下载 → 语言设置 → 启动主脚本  │
└───────────────────────┬──────────────────────────────┘
                        │
                        ▼
┌──────────────────────────────────────────────────────┐
│                  core/main.sh (主控)                   │
│         交互式菜单驱动 → 调度各功能模块                   │
└──┬──────┬──────┬──────┬──────┬──────┬────────────────┘
   │      │      │      │      │      │
   ▼      ▼      ▼      ▼      ▼      ▼
 menu   handler  check  read  share  generate
(菜单)  (处理器) (验证) (输入) (分享)  (生成器)
                │
                ├──► service/nginx.sh  (Nginx 编译安装)
                ├──► service/ssl.sh    (SSL 证书管理)
                ├──► service/docker.sh (Docker/WARP/Cloudreve)
                ├──► tool/traffic.sh   (流量统计)
                └──► tool/geodata.sh   (GeoIP/GeoSite 更新)
```

## 目录结构与职责

### 入口脚本

| 文件 | 职责 |
|------|------|
| `install.sh` | 项目安装引导脚本。负责系统检测（OS/版本）、依赖安装（curl/wget/jq/qrencode 等）、从 GitHub 下载项目文件、解析命令行参数（`--lang`、`--vision`、`--xhttp`、`--fallback`）、初始化配置目录（`.xray-script/`）、语言选择，最终启动 `core/main.sh` |

### 核心模块 (`core/`)

| 文件 | 职责 |
|------|------|
| `main.sh` | **主控脚本**。加载 i18n，提供交互式菜单驱动的流程控制。根据用户选择调度：一键安装、Xray 安装/卸载/启停、配置管理（协议切换、路由规则、SNI 配置、端口修改、GeoData 定时任务、语言切换）、分享链接展示、流量统计 |
| `handler.sh` | **核心处理器**（2225 行）。执行所有具体操作：Xray/Nginx 的安装与卸载、配置文件生成与模板替换、服务启停管理、X25519 密钥对管理、路由规则增删、SNI 端口检查、自定义站点管理、Nginx Stream 配置重建、CA 服务器切换等 |
| `menu.sh` | **菜单界面**。提供多级交互式菜单：主菜单、语言选择、一键安装、Xray 版本选择、协议配置、Web 配置、CA 供应商、配置管理、路由管理、SNI 配置、自定义站点管理。含随机 ASCII Banner 和状态展示 |
| `check.sh` | **输入验证器**。验证 IP 地址、端口范围、UUID 格式、密码强度、URL 路径、Short ID、域名安全性（格式+DNS 解析+TCP 连接+TLS 1.3+X25519）、DNS 解析到本机、Xray 版本存在性、邮箱格式、反代目标、自定义站点域名冲突、防火墙端口放行 |
| `generate.sh` | **随机生成器**。生成随机端口（1025-65535）、UUID（优先使用 `xray uuid`）、随机密码（16-64 位）、目标域名（从 `config.json` 的 target 池随机选取）、服务器名称列表、X25519 密钥对、Short ID（十六进制）、URL 路径 |
| `read.sh` | **用户输入读取**。通过参数映射表（`param_map`）将命令行选项映射到 i18n 提示文本，支持 config 和 rule 两种提示样式，返回用户输入内容 |
| `share.sh` | **分享链接生成**。根据当前配置（tag 类型），自动提取服务端参数，构造 VLESS/Trojan 协议的分享链接（含 Reality/TLS/mKCP/XHTTP 各种组合），生成二维码。SNI 模式下生成多达 5 组链接 |

### 服务管理模块 (`service/`)

| 文件 | 职责 |
|------|------|
| `nginx.sh` | 从源代码编译安装 Nginx，集成最新版 OpenSSL 和可选 Brotli 压缩模块。管理 systemd 服务配置，支持安装、更新、卸载操作 |
| `ssl.sh` | 使用 acme.sh 管理 SSL 证书。支持安装/更新/卸载 acme.sh，通过 HTTP-01 挑战签发/续期证书，支持 ZeroSSL 和 Let's Encrypt 两种 CA，管理证书部署到 Nginx |
| `docker.sh` | Docker 环境管理。安装 Docker、管理 Cloudflare WARP 代理容器、管理 Cloudreve v3/v4 私有网盘容器服务 |

### 工具模块 (`tool/`)

| 文件 | 职责 |
|------|------|
| `traffic.sh` | 通过 Xray API（`statsquery`）查询实时流量统计数据，按 Inbound/Outbound/User 维度展示上传/下载流量 |
| `geodata.sh` | 从 GitHub 下载最新的 GeoIP 和 GeoSite 数据库文件（来自 Loyalsoldier/v2ray-rules-dat），更新后自动重启 Xray |

### 配置文件 (`config/`)

| 目录/文件 | 职责 |
|------|------|
| `config.json`（根目录） | 项目元数据和模板配置。包含版本号、Xray 参数模板、Nginx 配置、目标域名池（用于 REALITY 伪装）、路由规则模板 |
| `config/xray/*.json` | Xray 配置模板文件：`Vision.json`、`XHTTP.json`、`mKCP.json`、`Trojan.json`、`Fallback.json`、`SNI.json` |
| `config/nginx/conf/` | Nginx 配置模板：主配置、Stream SNI 分流、站点配置、安全配置、限流配置、反向代理配置等 |
| `i18n/zh.json` / `en.json` | 国际化数据文件，提供中英文菜单文本、提示信息、错误描述等 |

### 运行时数据 (`.xray-script/`，运行时生成)

| 路径 | 职责 |
|------|------|
| `.xray-script/config.json` | 运行时主配置文件，存储所有当前配置状态（Xray 版本/tag/uuid/密钥/端口、Nginx 域名/CDN/CA、路由规则等） |
| `.xray-script/xray/config.json` | Xray 最终运行时配置文件（由模板+参数生成） |
| `.xray-script/acme.sh/` | acme.sh 安装目录和 SSL 证书数据 |
| `.xray-script/bin/xray` | Xray 二进制文件安装位置 |
| `.xray-script/geodata/` | GeoIP/GeoSite 数据库文件 |

## 核心工作流程

### 1. 安装流程

```
install.sh
  ├── 解析参数 (--lang, --vision, --xhttp, --fallback, -d)
  ├── 初始化 .xray-script/ 目录结构
  ├── 加载 i18n (中文/英文)
  ├── 检查 root 权限
  ├── 检查操作系统兼容性
  ├── 检查/安装系统依赖 (首次运行或 --check-deps)
  ├── 检查版本更新 (对比 GitHub config.json)
  ├── 语言选择 (首次运行时弹出菜单)
  └── 启动 core/main.sh (传递快速安装选项)
```

### 2. 主菜单流程

```
core/main.sh → processes_index()
  ├── 显示 ASCII Banner
  ├── 显示状态 (Xray 版本 / 配置标签 / WARP 状态)
  └── 主菜单选择:
      ├── 1. 一键安装 → processes_full_installation()
      ├── 2. Xray 安装 → processes_xray()
      ├── 3. 卸载 → handler --purge
      ├── 4-6. 启动/停止/重启 → handler --start/--stop/--restart
      ├── 7. 分享链接 → share.sh
      ├── 8. 流量统计 → traffic.sh
      └── 9. 配置管理 → processes_config()
           ├── Xray 配置切换
           ├── 路由规则管理
           ├── SNI 配置管理
           ├── 端口修改
           ├── GeoData 定时任务
           └── 语言切换
```

### 3. Xray 配置流程（以 Vision 为例）

```
用户选择协议 → handler.sh
  ├── exec_read() 读取用户输入 (端口/UUID/域名/Short ID 等)
  │   └── check.sh 验证每项输入
  ├── generate.sh 生成缺失参数 (UUID/X25519 密钥/Short ID 等)
  ├── handler_script_config() 更新 .xray-script/config.json
  ├── handler_x25519_config() 写入密钥对
  ├── handler_xray_config()
  │   ├── 读取对应模板 (config/xray/Vision.json)
  │   ├── 替换模板占位符 (UUID/端口/域名/密钥/ShortID 等)
  │   ├── 处理路由规则 (BT/CN/AD 屏蔽)
  │   └── 写入 .xray-script/xray/config.json
  ├── handler --install (安装 Xray 二进制)
  ├── handler --restart (重启 Xray 服务)
  └── share.sh (生成并展示分享链接+二维码)
```

### 4. SNI 模式流程（最复杂）

SNI 模式集成了 Nginx 反向代理和 SSL 证书：

```
SNI 配置 → handler.sh
  ├── check_sni_ports() 检查 80/443 端口和防火墙
  ├── 选择 CA (ZeroSSL / Let's Encrypt)
  ├── 安装 Nginx (从源码编译)
  ├── 签发 SSL 证书 (acme.sh + HTTP-01)
  ├── 配置 Nginx:
  │   ├── Stream 层 SNI 分流 (域名→Xray Vision / CDN→Nginx)
  │   ├── 站点配置 (反向代理/静态页面)
  │   └── 自定义站点管理
  ├── 配置 Xray (多 inbound: Vision + XHTTP)
  ├── 展示 5 组分享链接:
  │   ├── sni_vision_reality
  │   ├── sni_xhttp_reality
  │   ├── sni_tls_down (CDN 下行)
  │   ├── sni_xhttp_cdn (CDN 直连)
  │   └── sni_reality_down (Reality 下行)
  └── 可选: Cron 定时任务 (证书续签/Nginx 更新/GeoData 更新)
```

## 支持的协议配置

| 协议标签 | 传输方式 | 安全层 | 特点 |
|---------|---------|--------|------|
| **Vision** | TCP | REALITY | XTLS Vision 流控，高性能直连 |
| **XHTTP** | XHTTP | REALITY | 支持上下行分离，抗封锁能力强 |
| **mKCP** | KCP | 无 | 基于 UDP，适合高丢包网络 |
| **Trojan** | TCP/XHTTP | REALITY | Trojan 协议兼容 |
| **Fallback** | TCP+XHTTP | REALITY | 双 inbound，Vision+XHTTP 双链接 |
| **SNI** | TCP+XHTTP | REALITY+TLS | Nginx SNI 分流，支持 CDN，5 种链接组合 |

## 国际化机制

- 语言配置存储在 `.xray-script/config.json` 的 `language` 字段
- 支持 `zh`（中文）、`en`（英文）、`auto`（自动检测系统 `$LANG`）
- 每个脚本通过 `load_i18n()` 函数加载对应的 `i18n/{lang}.json` 文件
- `install.sh` 有独立的内嵌 i18n 关联数组（用于项目还未下载时的初始阶段）
- 菜单提示输出到 `stderr`（`>&2`），数据结果输出到 `stdout`，保证管道传递不冲突

## 配置管理核心

`config.json`（根目录模板）与 `.xray-script/config.json`（运行时）通过 `jq` 工具进行读写：

- **handler_script_config()** — 重置并写入 Xray 配置参数
- **handler_x25519_config()** — 生成并写入 X25519 密钥对
- **handler_xray_config()** — 从模板生成最终 Xray 运行时配置
- **handler_reset_script_config()** — 按区块（xray/nginx）重置字段，保留关键字段
- **reset_json_fields()** — 通用 JSON 字段重置工具函数

## 依赖关系

```
系统依赖: bash, curl, wget, git, jq, sed, awk, grep, openssl, qrencode, socat, dig
Xray: xray-core 二进制 (从 GitHub Releases 下载)
Nginx: 从源码编译 (自动下载 OpenSSL + 可选 Brotli)
SSL: acme.sh (自动安装到 .xray-script/acme.sh/)
Docker: 可选 (用于 WARP 代理和 Cloudreve 网盘)
```

## 快速安装命令

```bash
# 默认安装 (交互式菜单)
bash install.sh

# 快速安装指定协议
bash install.sh --vision
bash install.sh --xhttp
bash install.sh --fallback

# 指定语言
bash install.sh --lang=en

# 强制检查依赖
bash install.sh --check-deps
```
